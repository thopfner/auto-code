# UI Critic Rubric

## Highest-severity failures

- primary action is unclear or buried
- UI breaks at a common viewport
- major state is missing
- visual system drift is obvious on a shared surface
- accessibility or semantics likely block normal use

## Medium-severity failures

- spacing or typography hierarchy is inconsistent
- actions have weak affordance
- tables, forms, or cards do not match surrounding system density
- feedback states are technically present but visually weak

## Lower-severity failures

- polish inconsistencies
- minor alignment issues
- decorative choices that slightly dilute clarity

## Evidence handling

If screenshots or browser verification are absent, downgrade confidence and call that out explicitly.

## Rule

This skill should create pressure, not chaos. Report the strongest violations only and avoid flooding the workflow with stylistic nitpicks.
