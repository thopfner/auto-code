# Auto Forge Controller UI Tokens

Last refreshed: 2026-04-28

## Token Direction

Use semantic tokens from the first implementation pass. Avoid raw one-off colors in product UI.

Minimum token set:

- `--color-bg-canvas`
- `--color-bg-surface`
- `--color-bg-muted`
- `--color-text-primary`
- `--color-text-secondary`
- `--color-border-subtle`
- `--color-accent`
- `--color-success`
- `--color-warning`
- `--color-danger`
- `--space-*`
- `--radius-sm`
- `--radius-md`
- `--shadow-*`

## Visual Constraints

- Avoid one-note dark slate or purple dashboards.
- Cards should be used for repeated items, setup steps, task rows, and modals; do not nest cards.
- Use stable dimensions for status chips, buttons, tables, and log panes so live updates do not shift layout.
