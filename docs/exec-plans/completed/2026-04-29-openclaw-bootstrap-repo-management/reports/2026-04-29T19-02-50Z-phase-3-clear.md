## Accepted Behavior

- Repo-scoped Ed25519 key generation is implemented under `AUTO_FORGE_SSH_KEY_ROOT` or `/etc/auto-forge-controller/ssh`.
- Private keys are stored under restrictive repo-key directories and checked for mode `0600`.
- Telegram commands are present for `/repo key create`, `/repo key show`, `/repo key test`, `/repo key github-add`, and `/repo git-test`.
- Operator output exposes public key and fingerprint only; private-key material is not returned through Telegram/API responses.
- Git read validation uses `git ls-remote` with `GIT_SSH_COMMAND` pinned to the repo key for SSH remotes.
- Push validation uses `git push --dry-run` with the repo key.
- GitHub deploy-key API creation defaults to read-only and requires explicit `--write` for write access.
- GitHub deploy-key API calls use current GitHub REST deploy-key headers and payload shape.
