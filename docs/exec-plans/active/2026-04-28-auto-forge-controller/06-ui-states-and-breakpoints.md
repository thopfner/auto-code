# UI States And Breakpoints

## Required States

- Loading.
- Empty.
- Error.
- Success.
- Disabled.
- Long-content/overflow.
- Stale connection.
- Waiting for approval.
- Blocked external.
- Retrying.
- Cancelled.

## Breakpoints

- Mobile: onboarding and status views must be usable.
- Desktop: queue, task detail, and logs should use denser layouts.

## Refresh Behavior

Task status views must handle live refresh without layout jumps and must preserve readable historical events.
