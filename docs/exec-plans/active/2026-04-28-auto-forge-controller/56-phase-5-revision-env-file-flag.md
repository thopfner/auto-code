# Phase 5 Revision - Setup Wizard Env File Flag

Execution mode: `FINAL_SHIPGATE_REVISION`
Validation level: `FULL_REBUILD`

## Why This Revision Exists

QA found that the new setup wizard documents and implements `--env-file`, but the current `npm run setup:vps -- ...` path runs through `tsx`/Node. Node treats `--env-file` as a runtime option before the application can create the file. On a fresh VPS path such as `/etc/auto-forge-controller/auto-forge.env`, the command can fail before the wizard starts.

This breaks the fresh-VPS root-owned env-file flow required by `55-phase-5-revision-vps-setup-wizard.md`.

## Blocking Finding

Reproduction from `/var/www/html/auto.thapi.cc`:

```bash
tmpdir=$(mktemp -d /tmp/auto-forge-setup-qa-XXXXXX)
runuser -u tyler -- npm run setup:vps -- \
  --non-interactive \
  --public-base-url https://forge.example.com \
  --api-port 3000 \
  --web-port 5173 \
  --openclaw-base-url https://openclaw.example.com \
  --openclaw-token-ref env:OPENCLAW_TOKEN \
  --telegram-bot-token-ref env:TELEGRAM_BOT_TOKEN \
  --telegram-chat-id -100123 \
  --codex-auth-ref env:OPENAI_API_KEY \
  --env-file "$tmpdir/.env" \
  --setup-path "$tmpdir/setup.json"
```

Observed result:

```text
node: /tmp/.../.env: not found
```

The application never reaches `writeEnvBlock`, so it cannot create the operator-selected env file.

Finding type: `execution_miss`.

## Required Fix

- Replace the user-facing setup wizard option `--env-file` with a Node-safe flag name such as `--runtime-env-file`, `--env-output`, or `--auto-forge-env-file`.
- Update all code paths that read the env-file option:
  - interactive setup wizard
  - non-interactive setup wizard
  - CLI help text
  - deployment docs
  - reports or examples generated by the implementation
- Do not keep `--env-file` as the documented npm-run path. It is unsafe for this invocation shape.
- Optional compatibility may be supported only if it does not pass through Node/tsx as `--env-file`; otherwise omit compatibility and document the new flag.
- Add a deterministic test or script-level check proving the wizard can create a missing env file at an operator-selected path.
- Add a test or assertion that the selected env file is mode `0600` and the setup JSON still contains only secret references.
